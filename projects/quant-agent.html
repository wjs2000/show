双引擎融合逻辑

两个模块的分工不是并列展示，而是前后承接：量化模型负责排序，事件 Agent 负责解释、确认和风险复核。

量化主引擎

用 ETF 日线面板、46 个固定因子和多模型输出形成候选排序。

  • 预测未来 10 个交易日相对候选池中位数的强弱。
  • 输出候选 ETF、概率、策略目标和滚动调仓模拟。
  • 负责“先把可交易候选排出来”。
排序 + 解释确认催化 · 风险复核 · 决策观察

事件情绪 Agent

把新闻、政策、披露和社区评论转成可追溯的信息增强信号。

  • 生成事件对象、板块情绪和 ETF 信息增强画像。
  • 用于解释模型候选背后的信息变化。
  • 负责“说明为什么该看、哪里要谨慎”。

工作台展示

从预测候选到可解释观察

量化工作台输出候选 ETF 和模拟策略，事件 Agent 将当日公开信息映射到 ETF 主题上，形成“候选排序 + 信息画像 + 风险复核”的组合视图。

  • 模型输出解决“谁排在前面”。
  • 事件画像解释“当日信息为什么支持或削弱”。
  • 风险分用于识别需要人工复核的主题，避免只看正向概率。

事件 Agent 架构

规则主链路 + LLM 结构化增强

Agent 采用固定 Workflow：真实公开来源进入数据获取、清洗过滤、去重聚类、统一证据对象、可选 LLM 结构化、事件融合、板块情绪和 ETF 信息增强画像。

  • 数据源包括财经新闻、专业披露与会议纪要、基金社区评论等公开信息。
  • V2 事件本体统一映射政策、宏观、产业、披露、地缘、媒体和社区情绪。
  • LLM 失败时不阻断主流程，分析层按规则结果兜底完成。
